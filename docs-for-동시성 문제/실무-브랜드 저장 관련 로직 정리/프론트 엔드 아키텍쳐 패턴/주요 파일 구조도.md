# 프론트 재현용 주요 파일 구조도(권장)

## 권장 스택
- React + TypeScript + Vite
- 상태관리: Zustand 또는 React Query
- HTTP: Axios

## 권장 디렉토리
```text
tx-sync-lab-frontend/
  src/
    app/
      router.tsx
      queryClient.ts
    pages/
      BrandSyncPage.tsx
      SyncFailListPage.tsx
    features/brand-sync/
      api/
        brandApi.ts
      model/
        types.ts
      ui/
        BrandSyncForm.tsx
        BrandGrid.tsx
        SyncStatusBadge.tsx
        RetryButton.tsx
    shared/
      lib/http.ts
      ui/
      utils/
```

## 백엔드 연동 포인트(브랜드)
- `POST /api/airstar/brands/partner-brands-save`
- `GET /api/airstar/brands/partner-brands`
- `GET /api/airstar/brands/platform-brands`

## 화면 최소 구성
1. 브랜드 저장 요청 화면
2. sync 상태(대기/성공/실패) 목록 화면
3. 실패 건 재시도 화면

## 재현 테스트 UX(동시성 실험용)
- `동일 brandMid 2회 동시 저장` 버튼
- `외부 API 지연 시뮬레이션` 토글
- `실패건만 필터` + `일괄 재시도`
