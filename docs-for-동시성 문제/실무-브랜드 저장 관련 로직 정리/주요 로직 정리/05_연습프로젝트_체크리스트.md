# 05. 연습 프로젝트 구현 체크리스트

## 목표
외부 API 연동 실패/지연 상황에서 DB 정합성을 잃지 않는 구조를 직접 구현

## 백엔드(스프링부트) 체크
- rand 테이블에 sync 관련 컬럼 추가
- 저장 API는 PENDING까지 저장 후 커밋
- 커밋 이후 외부 호출 서비스 분리
- 실패 건 재시도 API 또는 스케줄러 구현
- 중복 방지(idempotency key 또는 unique + 상태 전이 규칙)

## 외부 API 모의 서버 체크
- 성공/실패/타임아웃 응답 토글 가능하게 구현
- 같은 요청 재수신 시 중복 처리 규칙 정의

## 프론트(관리 화면) 체크
- 요청 성공/실패 표시
- sync_status 필터(성공/실패/대기)
- 실패 건 재시도 버튼

## 테스트 케이스
- 정상 저장 + 외부 성공
- DB 성공 + 외부 실패
- 외부 타임아웃 + 실제 외부 반영
- 동일 키 동시 요청(2개 이상)
