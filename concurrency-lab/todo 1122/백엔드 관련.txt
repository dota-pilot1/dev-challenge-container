===========================================
 관리자 백엔드 프로젝트 장악 로드맵
===========================================

[Phase 1] 전체 그림 파악 (제일 먼저)
-------------------------------------------
1. 메뉴 / 라우팅 구조 분석
   - 어떤 페이지가 있는지, URL 구조가 어떤지
   - 이 시스템이 "뭘 하는 건지" 한눈에 보임

2. DB 테이블 관계도 (ERD) 그리기
   - 테이블 간 FK, 연관관계 파악
   - DBeaver ERD 기능 활용

3. API 목록 정리
   - Controller 기준으로 URL → 기능 매핑표 작성


[Phase 2] 코드 깊이 파악
-------------------------------------------
4. 아키텍처 패턴 분석
   - Controller → Service → Mapper 흐름 파악
   - 공통 모듈, 유틸 클래스 확인

5. 공통 처리 구조 파악
   - 인증/인가 (Spring Security, 인터셉터 등)
   - 예외 처리 구조
   - 공통 응답 포맷

6. 복잡한 SQL 모조리 분석
   - 조인 많은 쿼리, 서브쿼리, 동적 SQL
   - 여기가 버그 핫스팟

7. 설정 파일 분석
   - application.yml
