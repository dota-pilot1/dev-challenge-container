# 01. 전체 요청 흐름 요약

## 대상 API
- POST /api/airstar/brands/partner-brands-save
- 위치: src/main/java/com/cj/freshway/fs/cps/airstar/brand/AirstarBrandController.java

## 단계별 흐름
1. 클라이언트가 브랜드 목록(List<PartnerBrandReqDto>)을 전송
2. 컨트롤러가 AirstarBrandService.savePartenrBrands(...) 호출
3. 서비스에서 상태값(C,U) 기준으로 내부 DB INSERT/UPDATE 수행
4. 같은 서비스 메서드 안에서 외부 연동용 payload를 구성
5. MiddlewareApiManager를 통해 중계 서버로 POST 전송
6. 중계 서버가 외부 Airstar API(CP204/CP206) 호출
7. 응답 수신 후 메서드 종료

## 관련 인터페이스 ID
- CP204: 파트너 브랜드 등록
- CP206: 파트너 브랜드 수정

## 서버 역할
- BO 백엔드: 내부 DB 변경 + 연동 요청 오케스트레이션
- 중계 서버(GWMS): destination 정보 기반 외부 API 프록시 호출
- 외부 플랫폼(Airstar): 최종 브랜드 등록/수정 처리
