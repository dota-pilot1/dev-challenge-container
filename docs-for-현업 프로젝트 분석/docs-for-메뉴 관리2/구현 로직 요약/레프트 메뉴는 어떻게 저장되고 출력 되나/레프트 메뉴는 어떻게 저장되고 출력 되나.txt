# 레프트 메뉴(사이드바) 저장 및 출력 로직

## 1. 개요
레프트 메뉴(Left Menu)는 사용자가 헤더(Header)에서 선택한 **대메뉴(Lvl 3)**에 속하는 **중메뉴(Lvl 4)**와 **소메뉴(Lvl 5)**를 계층적으로 표시하는 영역입니다.

## 2. 데이터 흐름
1. **초기 로드**: 앱 시작 시 백엔드로부터 전체 메뉴 트리를 받아 `fwMenuState`(Recoil Atom)에 저장합니다.
2. **헤더 선택**: 사용자가 헤더의 대메뉴를 클릭하면, 해당 대메뉴가 `active` 상태가 되고, 이 대메뉴의 하위 트리(subMenus)가 `fwLeftMenu` 상태로 설정됩니다.
3. **렌더링**: `LeftMenu.jsx` 컴포넌트는 `fwLeftMenu` 데이터를 구독하여 화면에 그립니다.

## 3. 상세 출력 로직 (LeftMenu.jsx)

컴포넌트 내에서 다음과 같은 중첩 반복문을 통해 메뉴를 출력합니다.

### 1Depth (중메뉴, DB prog_lvl=4)
- 데이터 소스: `fwLeftMenu.subMenus`
- CSS 클래스: `left-menu-1depth`, `title-1depth`
- 역할: 소메뉴들을 그룹화하는 폴더 역할 또는 직접 실행 가능한 메뉴.

### 2Depth (소메뉴, DB prog_lvl=5)
- 데이터 소스: `item.subMenus` (중메뉴의 하위 리스트)
- CSS 클래스: `left-menu-2depth`, `title-2depth`
- 역할: 실제 클릭하여 해당 화면으로 이동하는 실행 메뉴.

### 3Depth (추가 Depth, 현재 미사용 추정)
- 데이터 소스: `subItem.lastMenus`
- 코드 상에는 존재하나 백엔드 로직에서는 Lvl 5까지만 조회하므로 실제로는 데이터가 없거나 확장성을 위해 존재함.

## 4. 상태 관리 (Recoil & Hooks)
- **useFwTab Hook**: `fwLeftMenu`, `handleActiveLeftMenu` 등을 제공하여 컴포넌트와 상태를 연결합니다.
- **fwLeftMenuOpenAtom**: 사이드바의 열림/닫힘(Toggle) 상태를 관리합니다.

## 5. 개발자용 메뉴 (Hardcoded)
- `devMenuList` 배열을 통해 DB에 없는 개발용/테스트용 메뉴(예: '인천공항 기준정보관리')를 하드코딩으로 추가하여 렌더링하고 있습니다.
